UI Automation Project — Automation Test Store (SDET Practicum)
Данный проект представляет собой выполнение технического задания по автоматизации тестирования UI. В рамках работы автоматизированы ключевые пользовательские сценарии интернет-магазина с применением современных практик разработки автотестов.

Стек технологий
Язык: Java 17

Фреймворк: JUnit 5

Инструмент автоматизации: Selenium WebDriver

Сборка: Maven


Паттерны: Page Object Model (POM), Driver Factory (ThreadLocal), Singleton.

Описание тест-кейсов
1. Фильтрация и сортировка в категориях
Цель: Проверить логику сортировки товаров по наименованию и стоимости.
Ключевые проверки:Сортировка по имени (A-Z и Z-A).

Сортировка по цене (возрастание и убывание).

Реализовано через программное сравнение списков (Assertions).

2. Поиск и работа с корзиной
Цель: Проверить точность поиска, добавление конкретных позиций и корректность пересчета итоговой суммы.
Сценарий:

Поиск по запросу "shirt".

Добавление в корзину 2-го и 3-го товаров из выдачи.

Динамический поиск самого дешевого товара в корзине и удвоение его количества.
Ожидаемый результат: Итоговая сумма (Total) обновляется в соответствии с изменениями.

3. Массовое управление корзиной
Цель: Проверить функционал удаления товаров по их порядковому номеру.
Сценарий: Добавление 5 случайных товаров и удаление всех четных по порядку позиций.
Ожидаемый результат: В корзине остаются только 1-й, 3-й и 5-й товары.


Инструкция по запуску проекта
1. Клонирование репозитория
Откройте терминал и выполните команду для скачивания проекта:
git clone https://github.com/whatislove24/SimbirSoftUI
2. Переход в корневую папку проекта
Перейдите в директорию, созданную после клонирования:
cd SimbirSoftUI
3. Запуск тестов
Для автоматического запуска всех тестов и сбора результатов используйте Maven:
mvn clean test
4. Генерация и просмотр Allure-отчета
После завершения тестов выполните команду для визуализации результатов в браузере:
mvn allure:serve

Архитектурные особенности
Параллелизация: Реализован параллельный запуск тестов с использованием ThreadLocal<WebDriver>, что обеспечивает изоляцию данных в потоках.
Стабильность: Использование WebDriverWait и стратегий обработки StaleElementReferenceException для борьбы с "флакующими" (нестабильными) тестами.
